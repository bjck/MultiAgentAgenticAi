## Overview

This `src/main/resources` directory contains **configuration and resource files** for the MultiAgent backend.

## Key Files & Folders

- `application.yml`: Primary Spring Boot configuration, including `multiagent.*` settings, model providers, and server ports.
- `db/`: Liquibase database migration files.
  - `db/changelog/01-*.xml`, `02-*.xml`, etc.: Incremental changelog files that define and evolve the PostgreSQL schema (sessions, orchestration logs, tool calls, external documents, MCP servers, etc.).
  - `db/changelog/db.changelog-master.xml`: Master changelog that includes the individual changelog files.
- (When present) `static/`: Static assets served directly by Spring Boot (typically the bundled frontend).

## Agent Notes

- Any change to the database schema should be expressed via a new Liquibase changelog file under `db/changelog` and wired into `db.changelog-master.xml`.
- Keep `application.yml` as the single source of truth for `multiagent.*` defaults; avoid hard‑coding configuration in Java.
- When updating bundled frontend assets, ensure they are placed under `static/` (usually generated by the build) rather than edited by hand.
